# 生信探癌公众号风格指南

## Positioning

`生信探癌` 面向生信、肿瘤、单细胞/空间组学方向的研究者、研究生和临床转化读者。文章要比组会笔记更易读，比泛科普更专业。

## Voice

- 用专业科普语气：准确、清楚、有阅读推进感。
- 少用空泛形容词，多写“这篇文章解决了什么问题”“这个图证明了什么”“这个分析能怎么复用”。
- 可以用短句增强节奏，但不要用夸张标题党。
- 中文为主，关键术语保留英文，例如 `scRNA-seq`、`spatial transcriptomics`、`CellChat`。

## Title Style

默认给 3-5 个标题候选，覆盖不同角度：

- 期刊机制型：`Cancer Discovery | 单细胞测序绘制<癌种>免疫全景图：<细胞/通路/机制轴>主导<表型>`
- 机制型：`Nature 子刊揭示：<细胞/通路>如何重塑<癌种>微环境`
- 技术型：`单细胞 + 空间组学解析<癌种>：从细胞状态到治疗线索`
- 问题型：`为什么<治疗/转移/耐药>会发生？这篇<癌种>研究给出新线索`
- 方法启发型：`这套<技术/分析流程>，值得用到你的<癌种/TME>课题里`

避免：

- 过度承诺疗效或临床转化。
- 没有论文依据的“首次发现”“颠覆认知”。
- 标题里堆太多方法名。

## Article Rhythm

- 开头 1-2 段必须说清楚：疾病场景、核心问题、这篇文献的亮点。
- 默认正文结构使用：`研究背景 -> 样本与方法 -> 结论详解 -> 生信方法看点 -> 小结 -> 原文引用与图源说明`。
- `研究背景` 要解释领域缺口，不要只说“某癌很常见”。
- `样本与方法` 要讲清队列、样本类型、核心技术和验证逻辑，不要写成 Methods 清单。
- `结论详解` 是文章主体，可以重组原文 Results，但每个重组结果都必须保留图文对应。
- 每个大段落只讲一个问题。
- 每个结果模块优先采用“研究问题 -> 使用方法/队列 -> 关键发现 -> 这说明什么 -> 可借鉴点”的顺序。
- 对复杂方法可以单独设置“方法看点”，但不要把公众号文章写成软件教程，除非文章类型就是 `技术教程`。

## Result-Figure Rhythm

- 每个 Result/结论详解模块必须配至少一张原文局部图块。
- 图必须来自 `$read-literature-pdf` 生成的 Markdown 报告中的已有图片路径，优先 `figures/`，兼容 `figures_panel_modules/`。
- 不使用整页 PDF 截图、整页 Figure 或低清大图来凑图。
- AI 示意图可以帮助讲机制，但不能替代 Result 必需的原文数据图。
- 图片后必须紧跟 `图示解读` 和 `图源说明`，让读者知道“看哪里、说明什么、图从哪里来”。
- 如果 Markdown 报告没有某个 Result 的局部图块，正文必须提示：`该 Result 缺少可用局部图块，需回到 read-literature-pdf 阶段补图。`

## Backend Draft Rhythm

当用户说“做成公众号草稿”“上传到公众号”“放到后台”时，交付目标不是一个本地 HTML，而是微信公众平台后台里已经保存的草稿。

- `wechat.html` 只是中间稿，用于排版和粘贴；不能当作后台草稿交付。
- 正文图片必须通过微信编辑器或官方接口上传，变成微信图床图片；本地路径、`file://`、相对路径和 data URL 都不算成功。
- 公众号正文不能只剩文字。每个 Result 图块、AI 示意图和封面图都要在后台可见，或在 `wechat_draft_status.md` 写明失败原因。
- 封面图要优先设置到封面选择器；如果界面阻挡自动化，至少上传为正文第一张图，并清楚写出人工确认步骤。
- 保存后要记录：标题、正文图片数量、微信图床图片数量、封面状态、是否已保存、仍需人工检查什么。
- 永远不要在自动化流程里点击发表、发布、群发或不可逆确认按钮。

## Figure Captions

原文图：

```markdown
**图示解读：** Figure 2A-D 主要展示……
**图源说明：** 原文 Figure 2A-D，仅用于文献解读，请发布前确认版权/转载要求。
```

AI 图：

```markdown
**AI 示意图说明：** 本图为根据原文信息生成的辅助示意图，不是原文数据图。
```

## Terminology

- `tumor microenvironment`：肿瘤微环境，首次可写 `肿瘤微环境（TME）`。
- `single-cell RNA-seq`：单细胞 RNA 测序，常用 `scRNA-seq`。
- `spatial transcriptomics`：空间转录组。
- `cell-cell communication`：细胞通讯或细胞互作。
- `trajectory / pseudotime`：轨迹分析 / 拟时序分析。
- `copy number variation`：拷贝数变异，常用 `CNV`。

## Classification Tone

分类是发布辅助信息，不要在正文里生硬展示。可以在文末用一句自然的话承接：

`这篇文章更适合归入「肺癌」合集，同时打上「单细胞测序」「肿瘤微环境」「免疫治疗」等标签。`
